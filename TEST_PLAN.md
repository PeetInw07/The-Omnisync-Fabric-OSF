# 📑 Detailed Test Plan: OSF Quality Assurance 

## 1. Functional & Boundary Testing (การทดสอบฟังก์ชันและค่าขอบเขต)

* **เป้าหมาย:** ตรวจสอบความแม่นยำของอัลกอริทึมการเลือกเส้นทาง (Decision Logic)
* **ตารางกรณีทดสอบ (Test Cases):**

| กรณีที่ | กรณีทดสอบ | ค่าที่ป้อน (Tie Score) | ผลที่คาดหวัง | สถานะ |
| --- | --- | --- | --- | --- |
| 1 | **Strong Tie** | 0.9 | ระบบต้องเลือก High-Speed Lane | Pass ✅ |
| 2 | **Standard Tie** | 0.4 | ระบบต้องเลือก Standard Lane | Pass ✅ |
| 3 | **Boundary Low** | 0.0 | ระบบต้องทำงานได้ปกติในเลน Standard | Pass ✅ |
| 4 | **Boundary High** | 1.0 | ระบบต้องทำงานได้สูงสุดในเลน High-Speed | Pass ✅ |
| 5 | **Threshold Edge** | 0.7 | ระบบต้องเปลี่ยนเข้าสู่ High-Speed Lane (จุดตัด Logic) | Pass ✅ |

## 2. Negative Testing & Error Handling (การทดสอบกรณีข้อมูลผิดพลาด)

**เป้าหมาย** : เพื่อยืนยันความเสถียรของระบบ (System Resilience) เมื่อต้องรับข้อมูลที่ผิดรูปแบบหรือไม่เป็นไปตามเงื่อนไข ป้องกันโปรแกรมหยุดทำงาน (Crash)
| กรณีที่ | กรณีทดสอบ | ค่าที่ป้อน  | ผลที่คาดหวัง | สถานะ |
| --- | --- | --- | --- | --- |
| 1 | **Invalid Character** | "abc" | ระบบต้องไม่ Crash และทำการ Fallback ไปใช้ค่า 0.5 อัตโนมัติ| Pass ✅ |
| 2 | **Null Input** | (ค่าว่าง)| ระบบต้องจัดการค่าว่างและดำเนินการต่อโดยใช้ค่ามาตรฐาน | Pass ✅ |
| 3 | **Out of Range Value** | 99 | ระบบประมวลผลตาม Logic โดยไม่หยุดทำงาน | Pass ✅ |

## 3. Security Testing: Mind-Wall Firewall (การทดสอบความปลอดภัย)

**เป้าหมาย**: เพื่อยืนยันว่าข้อมูลอารมณ์ซึ่งเป็นข้อมูลส่วนบุคคล จะถูกส่งผ่านกระบวนการตรวจสอบความปลอดภัยที่เข้มงวด และป้องกันการเข้าถึงจากช่องทางที่ไม่ได้รับอนุญาต
 ข้อมูลที่ส่งถึงปลายทางต้องตรงกับที่รับมาจาก Layer 1 (ไม่มีการบิดเบือนระหว่างทาง)
| กรณีที่ | กรณีการทดสอบ | วิธีการทดสอบ (Methodology) | ผลลัพธ์ที่คาดหวัง | สถานะ |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Unauthorized Bypass | จำลองการข้ามขั้นตอนจาก Layer 1 ไป Layer 5 | ระบบต้องไม่อนุญาตให้ Output ทำงานหากไม่ผ่านการ Check ใน Layer 4 | Pass ✅ |
| 2 | Data Integrity Check | ตรวจสอบค่าอารมณ์ระหว่าง Input และ Output | ข้อมูลอารมณ์ปลายทางต้องตรงกับที่รับมาจาก Layer 1 100% | Pass ✅ |
| 3 | Firewall Persistence | รันโปรแกรมซ้ำเพื่อเช็คความเสถียรของสถานะ | ระบบต้องแสดงสถานะ "Mind-Wall Protection Active" ทุกครั้ง | Pass ✅ |

## 4. Performance & Sequential Testing (การทดสอบประสิทธิภาพ)
**เป้าหมาย**: เพื่อตรวจสอบว่าโครงสร้างการทำงานแบบลำดับชั้น (Layered Architecture) ทำงานได้อย่างถูกต้อง และจำลองระยะเวลาการประมวลผล (Latency) ให้สอดคล้องกับทฤษฎี
###
| กรณีที่ | กรณีการทดสอบ | เกณฑ์การตัดสิน (KPI) | ผลลัพธ์ที่คาดหวัง | สถานะ |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Sequential Order | ตรวจสอบการไหลของข้อมูล 1 -> 2 -> 3 -> 4 -> 5 | ข้อมูลต้องทำงานตามลำดับชั้น ห้ามข้ามเลเยอร์หรือสลับลำดับ | Pass ✅ |
| 2 | Simulated Latency | ตรวจสอบการทำงานของฟังก์ชัน `time.sleep(1)` | ระบบต้องมีความหน่วงคงที่เพื่อจำลองการประมวลผลจริงในแต่ละชั้น | Pass ✅ |
| 3 | Memory Stability | รัน Simulator ต่อเนื่อง 10 ครั้ง | โปรแกรมต้องทำงานได้ราบรื่น ไม่เกิดอาการค้างหรือ Infinite Loop | Pass ✅ |



---

## 5. User Acceptance Simulation (การยอมรับจากผู้ใช้)

* **เป้าหมาย:** ตรวจสอบว่าผลลัพธ์ที่ได้สื่อสารกับผู้ใช้ได้เข้าใจง่าย
### 
| กรณีที่ | กรณีการทดสอบ | จุดตรวจสอบ (Checkpoint) | ผลลัพธ์ที่คาดหวัง | สถานะ |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Feedback Clarity | ตรวจสอบข้อความสรุปผล (RESULT) | ต้องระบุประเภทอารมณ์และสถานะความปลอดภัยชัดเจนในบรรทัดเดียว | Pass ✅ |
| 2 | Instruction Clarity | ตรวจสอบคำแนะนำในการกรอกข้อมูล | ผู้ใช้ต้องเข้าใจว่าต้องกรอกค่าตัวเลขในช่วง 0.0 - 1.0 | Pass ✅ |
| 3 | Log Completeness | ตรวจสอบส่วน System Remark ใน Layer 3 | ต้องมีการบอกเหตุผลในการเลือกเส้นทาง (เช่น เพราะเป็น Strong Tie) | Pass ✅ |
